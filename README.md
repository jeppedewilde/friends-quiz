# API

## Opdracht
Maak een website met minimaal één content API en minimaal twee Web API's.

## TMDB API
Basis URL > https://api.themoviedb.org/3
Pad episode details > /tv/{series_id}/season/{season_number}/episode/{episode_number}
API key > ?api_key=${apiKey}
{series_id} = 1668
{season_number} = ${randomSeizoen}
{episode_number} = ${randomAflevering}

## Ideeen
### Idee 1
No excuses > Kies een reden waarom je niet wil gaan sporten en een coach schreeuwt een motiverende/vernederende/grappige speech

### Idee 2
Friends quiz!
TMDB API voor content, web socket API om met vrienden te spelen.
Game modes: Seizoen/aflevering raden, quote raden, 'wie zei dit' raden.

### Wat heb ik gedaan? / Hoe lang heb ik daarover gedaan?
Spelersnamen opslaan in array & deze tonen op het host scherm, browser id aan de naam koppelen, geen lege of dubbele namen accepteren.

Vragen generator gebouwd voor 3 type vragen: 1 foto + 4 mogelijke afleveringstitels, 1 afleveringstitel + 4 mogelijke afbeeldingen. 

### Wat heb ik geleerd?

### Wat ga ik morgen doen?

## Donderdag 16 april
### Wat heb ik gedaan? / Hoe lang heb ik daarover gedaan?
Host scherm laat de vragen zien en op play schermen geven de spelers antwoord.

Server telt mee hoeveel spelers antwoord hebben gegeven, kijkt de antwoorden na en deelt punten uit > zodra iedereen heeft geantwoord verschijnt het goede antwoord op het host scherm.

Heel veel errors en bug fixes tot dit allemaal werkte.

### Wat heb ik geleerd?
Ik ben weer dieper in de web socket API stof gedoken: gezorgd dat de flow van het spel met antwoorden geven werkt en de vragen nagekeken worden op de server.

### Wat ga ik de volgende keer doen?
De volgende keer moet ik nog wat bugs fixen in de flow van het spel (vooral bij vraagtype seizoen raden) en ga ik verder werken aan het vervolg van de flow.
